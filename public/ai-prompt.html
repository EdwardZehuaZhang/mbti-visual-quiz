AI Prompt Used to Generate Sequence Diagram

Assignment 2 · ID2116 Computing for Design · Personality Quiz Sequence Diagram
Tool used: Claude Opus (Anthropic) via Claude Code

Prompt

Create a clean UML sequence diagram for a web app called MBTI Visual Quiz. Use the MVC architecture with these participants, grouped clearly by layer: - User (actor, outside MVC) - View — React frontend running in the browser (Next.js, pages: /quiz and /results) - Controller — Next.js serverless API routes running on the server (/api/scene, /api/interpret, /api/results, /api/images) - OpenAI API — external AI service (gpt-4o-mini for most calls, gpt-4.5/gpt-4o for final results) - Image APIs — external image search (tries Pinterest, then Pexels, then Unsplash as fallback) The flow: 1. User opens the quiz. The View initialises a PersonalityState object in browser memory (no database — all state is client-side). 2. Loop (up to 11 rounds, exits early when personality is determined): - View calls /api/scene with the current state → Controller asks OpenAI to generate a scene description and a photo search keyword → returned to View - View calls /api/images with the keyword → Controller fetches 4 photos from Image APIs → returned to View - View shows the scene + 4 photos to the User - User taps one photo - View calls /api/interpret with the chosen photo and current state → Controller asks OpenAI what the choice reveals about the user's personality → OpenAI returns updated personality scores and a one-line insight → View saves the updated scores and shows the insight to the User 3. View calls /api/results with all choices and final scores → Controller asks OpenAI to determine the MBTI type and write a personalised paragraph → returned to View → View shows the result page to the User Formatting rules: - Solid arrows (→) for requests. Dashed arrows (-->) for responses. Every arrow must have an arrowhead. - Group the View, Controller, and External APIs in clearly labelled coloured boxes so the MVC structure is visually obvious. - Keep labels short and in plain English — no technical jargon. - Number each step. - Output ONLY a Mermaid sequence diagram code block — nothing else, no explanation.

How it was used

The prompt above was submitted to Claude Opus (claude-opus-4-5) via Claude Code. The model returned a Mermaid sequence diagram code block. The code was then embedded in an HTML page using Mermaid.js (CDN) and deployed to Vercel. Live diagram: https://mbti-visual-quiz.vercel.app/diagram
