  Note over User,View: App loads — state is set up in the browser
  User->>View: Open /quiz page
  View->>View: Create PersonalityState
(4 scores all start at 0)

  loop Up to 11 rounds — stops early once all 4 scores are confident enough

    Note over View,OpenAI: Step 1 — Generate a scene for this round
    View->>Ctrl: POST /api/scene
(sends current scores)
    activate Ctrl
    Ctrl->>OpenAI: "Design a visual scene that tests
introvert vs. extrovert" (or another trait)
    activate OpenAI
    OpenAI-->>Ctrl: Scene description + photo search keyword
    deactivate OpenAI
    Ctrl-->>View: Scene text + keyword returned
    deactivate Ctrl

    Note over View,Imgs: Step 2 — Fetch 4 photos for the user to choose from
    View->>Ctrl: GET /api/images?q=keyword
    activate Ctrl
    Ctrl->>Imgs: Search photos by keyword
    activate Imgs
    Imgs-->>Ctrl: 4 photos returned
    deactivate Imgs
    Ctrl-->>View: 4 images ready to display
    deactivate Ctrl

    Note over User,View: Step 3 — User picks a photo
    View->>User: Show scene description + 4 photos
    User->>View: Tap favourite photo

    Note over View,OpenAI: Step 4 — Interpret what the choice reveals
    View->>Ctrl: POST /api/interpret
(sends scene + chosen photo + current scores)
    activate Ctrl
    Ctrl->>OpenAI: "What does choosing this photo suggest
about the user's personality?"
    activate OpenAI
    OpenAI-->>Ctrl: Updated personality scores + one-line insight
    deactivate OpenAI
    Ctrl-->>View: New scores + insight text
    deactivate Ctrl
    View->>View: Save updated scores to PersonalityState
    View->>User: Show insight ("You seem drawn to...")

  end

  Note over View,OpenAI: Final step — determine the MBTI type
  View->>Ctrl: POST /api/results
(sends all choices + final scores)
  activate Ctrl
  Ctrl->>OpenAI: "Based on all choices, what is the MBTI type?

  activate OpenAI
  Note right of OpenAI: Uses best available model:
gpt-4.5 → gpt-4o (fallback)
  OpenAI-->>Ctrl: MBTI type (e.g. INFP) + paragraph

How to read this diagram

Solid arrow →  A request or action being sent (e.g. calling an API)
Dashed arrow -->  A response being returned (e.g. data coming back)
Green box (activation)  Shows when a component is actively processing a request

Where is the data stored?

There is no database. The personality scores (PersonalityState) live entirely in the browser's memory. Each API call receives the current scores as part of the request body, and sends back updated scores in the response.

When does the loop stop?

The quiz runs for up to 11 rounds. It stops early if all four personality trait scores reach high confidence (above 75%). The four traits tested are: E/I · N/S · F/T · P/J.

What is the MVC pattern here?

View = the React components the user interacts with in the browser.
Controller = the Next.js API routes that handle requests and call external APIs.
Model = PersonalityState — the data structure that tracks scores (lives in the View layer since there's no server-side database).

What does OpenAI do?

OpenAI is called 3 times per round:
① /api/scene — creates the visual scene prompt
② /api/interpret — updates personality scores based on the chosen photo
③ /api/results — generates the final MBTI type and personalised paragraph
