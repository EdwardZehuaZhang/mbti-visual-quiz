sequenceDiagram
    actor User

    box rgb(219,234,254) View — Browser (Next.js)
        participant V as View
/quiz · /results
    end

    box rgb(220,252,231) Controller — Server (Next.js API Routes)
        participant C as Controller
/api/scene · /api/interpret
/api/results · /api/images
    end

    box rgb(254,243,199) External APIs
        participant AI as OpenAI API
(gpt-4o-mini / gpt-4.5)
        participant IMG as Image APIs
(Pinterest → Pexels → Unsplash)
    end

    User->>V: 1. Open quiz
    V->>V: 2. Initialise PersonalityState in memory

    loop Up to 11 rounds (exits early when personality determined)
        V->>C: 3. POST /api/scene — send current state
        C->>AI: 4. Generate scene description + photo keyword
        AI-->>C: 5. Scene text + keyword
        C-->>V: 6. Return scene + keyword

        V->>C: 7. GET /api/images — send keyword
        C->>IMG: 8. Fetch 4 photos (Pinterest → Pexels → Unsplash)
        IMG-->>C: 9. Return 4 photos
        C-->>V: 10. Return 4 photos

        V->>User: 11. Show scene + 4 photos
        User->>V: 12. Tap a photo

        V->>C: 13. POST /api/interpret — send chosen photo + current state
        C->>AI: 14. Analyse choice, update personality scores
        AI-->>C: 15. Updated scores + one-line insight
        C-->>V: 16. Return updated scores + insight
        V->>V: 17. Save updated scores to PersonalityState
        V->>User: 18. Show insight
    end

    V->>C: 19. POST /api/results — send all choices + final scores
    C->>AI: 20. Determine MBTI type + write personalised paragraph
    AI-->>C: 21. MBTI type + personalised paragraph
    C-->>V: 22. Return MBTI result
    V->>User: 23. Show results page
