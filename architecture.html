User Flow & API Sequence
How a quiz session progresses end-to-end
STEP 1 — PAGE LOAD
User visits /quiz. React initialises PersonalityState with all signals & confidence = 0.
STEP 2 — SCENE GENERATION
POST /api/scene
Sends current PersonalityState to GPT-4o-mini.
Returns: scene prompt, imageQuery keyword, target axis, intent.
STEP 3 — IMAGE FETCH
GET /api/images?q=<imageQuery>
Tries Pinterest → Pexels → Unsplash → Picsum.
Returns 4 shuffled images for the user to choose from.
STEP 4 — USER CHOOSES
User taps one of the 4 images. The chosen image's description & alt text are captured.
STEP 5 — INTERPRET CHOICE
POST /api/interpret
Sends scene + chosen image + full state.
Returns updated signals, confidence, 1-2 sentence interpretation.
STEP 6 — REPEAT OR FINISH
Repeat steps 2–5 until: 11 turns reached OR all 4 confidence scores > 0.75.
STEP 7 — FINAL RESULTS
POST /api/results
Sends full choice history + signals.
Returns MBTI type, personalised paragraph, trait breakdown.

OpenAI API — Prompts & Responses
What is sent to the API and what comes back for each endpoint
POST /api/scene  →  Model: gpt-4o-mini · temp 0.9 · max_tokens 200
SYSTEM (psychologist role)
Design visual MBTI quiz scenes
Pick ONE specific subject (2-3 words)
4 images = same subject, different versions
Target axis: EI / SN / TF / JP
Output JSON: scene, imageQuery, axis, intent
USER MESSAGE (per turn)
Current EI/SN/TF/JP signal values
Current confidence per axis
Turn number (of 11)
Lowest-confidence axis to target
Last 3 interpretations (context)
RESPONSE → JSON
scene — evocative prompt shown to user
imageQuery — search keyword (e.g. "tabby cat")
axis — MBTI axis this round probes
intent — what each choice reveals
POST /api/interpret  →  Model: gpt-4o-mini · temp 0.7 · max_tokens 300
SYSTEM (psychologist role)
Interpret image choices for MBTI signals
Signals: –1.0 (I/S/T/J) to +1.0 (E/N/F/P)
Adjust incrementally ±0.05–0.20 per choice
Confidence increases ±0.05–0.15 per turn
Be nuanced — avoid simplistic mappings
USER MESSAGE (per turn)
Scene that was presented
Chosen image: description + alt text
Current EI/SN/TF/JP signal values
Current confidence values
Turn number + last 3 interpretations
RESPONSE → JSON
signals — updated {EI, SN, TF, JP} floats
confidence — updated {EI, SN, TF, JP} floats
interpretation — 1-2 sentence insight
POST /api/results  →  Model: gpt-5.4 → gpt-4.5-preview → gpt-4o (fallback chain) · temp 0.8 · max_tokens 800
SYSTEM (psychologist role)
Determine final MBTI type from signals
Write rich, personalised 4-6 sentence paragraph
Reference specific choices the user made
Warm, affirming — not generic boilerplate
Include trait breakdown with axis scores
USER MESSAGE (final)
Final signal values for all 4 axes
Final confidence per axis
Full choice history (all turns): scene + image + insight
RESPONSE → JSON
type — e.g. "INFP", "ESTJ"
paragraph — personalised description
traitBreakdown — {EI, SN, TF, JP} label + score

PersonalityState — Core Data Model
Lives in client memory. Passed to every API call. Never persisted to a database.
TypeScript Interface
signals: { EI, SN, TF, JP } // –1.0 … +1.0
confidence: { EI, SN, TF, JP } // 0.0 … 1.0
choices: Array<{
  scene: string
  imageDescription: string
  interpretation: string
}>
turn: number // 0 → 11
selectedImages: Array<{url, pinId}>
Quiz Completion Logic
Quiz ends when either condition is true:

A All 4 confidence scores exceed 0.75
B Turn counter reaches 11 (hard limit)

Axis: EI (–=Intro, +=Extra) · SN (–=Sensing, +=iNtuition)
       TF (–=Thinking, +=Feeling) · JP (–=Judging, +=Perceiving)

API Routes Summary
All routes are stateless Next.js serverless functions deployed on Vercel
POST
/api/scene
Generates next quiz round. Takes PersonalityState, targets the lowest-confidence axis, returns scene + imageQuery for image search.
GET
/api/images
Fetches 4 shuffled images for a keyword query. Waterfall: Pinterest → Pexels → Unsplash → Picsum. Supports orientation and pinId params.
POST
/api/interpret
Interprets the user's image choice. Incrementally updates EI/SN/TF/JP signals and confidence. Returns a 1-2 sentence personality insight.
POST
/api/results
Final MBTI type generation. Uses best available GPT model with fallback chain (gpt-5.4 → gpt-4.5-preview → gpt-4o). Returns type + personalised paragraph + trait scores.
